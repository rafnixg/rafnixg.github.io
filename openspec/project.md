# Project Context

## Purpose
This is a personal portfolio website for Rafael, showcasing skills, projects, and contact information. The goal is to provide a professional online presence, demonstrate technical abilities, and facilitate networking opportunities.

## Tech Stack
- HTML5 for structure
- CSS3 for styling and responsive design
- JavaScript (Vanilla ES6+) for interactivity
- GitHub Pages for hosting and deployment

## Project Conventions

### Code Style
- Use semantic HTML5 elements
- Follow BEM methodology for CSS class naming
- Write clean, readable JavaScript with modern ES6+ features
- Use consistent indentation (tabs or 2 spaces)
- Add comments for complex logic

### Architecture Patterns
- Static site architecture with no server-side components
- Modular CSS with separate stylesheets for different sections
- Vanilla JavaScript for DOM manipulation and interactivity
- Asset organization in dedicated folders (fonts, images)

### Testing Strategy
- Manual cross-browser testing (Chrome, Firefox, Safari, Edge)
- Responsive design testing on various screen sizes
- Accessibility testing with keyboard navigation and screen readers
- Performance testing for fast loading times

### Git Workflow
- Main branch for production-ready code
- Feature branches for new developments (feature/feature-name)
- Conventional commit messages (feat:, fix:, docs:, etc.)
- Pull requests for code review before merging

## Domain Context
Personal portfolio website in the web development domain, focusing on frontend technologies, UI/UX design, and professional presentation.

## Important Constraints
- Must remain a static site (no backend or server-side processing)
- Fast loading times (< 3 seconds)
- Mobile-first responsive design
- Accessibility compliance (WCAG 2.1 AA)

## External Dependencies
- GitHub Pages for hosting
- Social media platforms for contact links
- Font libraries (if any, check assets/fonts/)
